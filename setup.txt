1. Create the frontend React client with Vite

   Command:

   ```bash
   npm create vite@latest client -- --template react
   cd client
   npm install
   ```

   Installed dependencies:

   - `react`: Core React library used to build the frontend UI with reusable components.
   - `react-dom`: Connects React components to the browser DOM so the app can render in `index.html`.

   Installed dev dependencies:

   - `vite`: Frontend build tool and development server for fast local development.
   - `@vitejs/plugin-react`: Adds React support to Vite, including JSX handling and React Fast Refresh.
   - `eslint`: Static analysis tool used to catch JavaScript and React code issues.
   - `@eslint/js`: Provides the recommended base ESLint rules for JavaScript.
   - `eslint-plugin-react-hooks`: Enforces the rules of React Hooks.
   - `eslint-plugin-react-refresh`: Helps keep React Fast Refresh compatible with component exports.
   - `globals`: Provides predefined global variables for ESLint configuration.
   - `@types/react`: Type definitions for React, useful for editor IntelliSense and typed tooling.
   - `@types/react-dom`: Type definitions for React DOM, useful for editor IntelliSense and typed tooling.

2. Install Tailwind CSS for styling

   Command:

   ```bash
   cd client
   npm install tailwindcss @tailwindcss/vite
   ```

   Installed dependencies:

   - `tailwindcss`: Utility-first CSS framework used to style the React UI directly with Tailwind classes.
   - `@tailwindcss/vite`: Vite integration for Tailwind CSS so Tailwind can be processed during development and production builds.

   Setup changes:

   - Added Tailwind to the Vite plugins list in `vite.config.js`.
   - Added `@import "tailwindcss";` in `src/index.css` so Tailwind styles are available globally in the React app.

3. Install frontend UI, routing, animation, and state dependencies

   Command:

   ```bash
   cd client
   npm install framer-motion lucide-react react-router-dom zustand
   ```

   Installed dependencies:

   - `framer-motion`: Animation library used to add smooth transitions, motion effects, and interactive UI animations in React.
   - `lucide-react`: React icon library used for clean, customizable SVG icons throughout the UI.
   - `react-router-dom`: Routing library used to create client-side pages and navigation in the React app.
   - `zustand`: Lightweight state management library used to share app state across React components without heavy boilerplate.

4. Initialize the backend server package

   Command:

   ```bash
   mkdir server
   cd server
   npm init -y
   ```

   Setup changes:

   - Created `server/package.json` to manage backend scripts, dependencies, package metadata, and Node.js module settings.
   - Added `"type": "module"` so backend JavaScript files can use modern ES module `import` and `export` syntax.

   Installed dependencies:

   - No backend npm dependencies were installed in this step.

5. Install backend API dependencies

   Command:

   ```bash
   cd server
   npm install express mongoose dotenv cors helmet
   npm install -D nodemon
   ```

   Installed dependencies:

   - `express`: Web framework used to create the backend API server, routes, middleware, and request/response handling.
   - `mongoose`: MongoDB object modeling library used to define schemas, models, and database queries.
   - `dotenv`: Loads environment variables from a `.env` file for values like database URLs, ports, and secrets.
   - `cors`: Enables Cross-Origin Resource Sharing so the frontend client can safely call the backend API from a different origin.
   - `helmet`: Adds common security-related HTTP headers to help protect the Express app.

   Installed dev dependencies:

   - `nodemon`: Automatically restarts the backend server during development when files change.

   Setup changes:

   - Added `start` script to run the backend with Node.js.
   - Added `dev` script to run the backend with Nodemon during development.

6. Install backend authentication dependencies and configure environment variables

   Command:

   ```bash
   cd server
   npm install bcryptjs jsonwebtoken
   ```

   Installed dependencies:

   - `bcryptjs`: Used to hash user passwords before storing them and compare login passwords securely.
   - `jsonwebtoken`: Used to create and verify JWT tokens for authenticated API requests.

   Environment setup:

   - Created `server/.env` for backend environment variables.
   - Added `PORT` to configure which port the Express server runs on.
   - Added `MONGO_URI` to store the MongoDB connection string outside the source code.
   - Added `JWT_SECRET` to sign and verify authentication tokens.
   - Added `.env` and `.env.*` to `server/.gitignore` so local secrets are not committed.

   Backend setup changes:

   - Created database connection setup in `src/config/db.js` using `mongoose.connect(process.env.MONGO_URI)`.
   - Updated `index.js` to load environment variables with `dotenv.config()` and connect to MongoDB before starting the API server.
   - Created `src/models/User.js` with a Mongoose user schema for account data, subscription tier, generation count, and timestamps.

7. Add authentication API routes and request rate limiting

   Command:

   ```bash
   cd server
   npm install express-rate-limit
   ```

   Installed dependencies:

   - `express-rate-limit`: Middleware used to limit repeated requests from the same IP address and reduce abuse of API endpoints.

   Backend setup changes:

   - Added global rate limiting in `index.js` with a 15 minute window and a 100 request limit per IP.
   - Mounted authentication routes under `/api/auth` in `index.js`.
   - Created `src/controllers/authController.js` with register, login, and profile controller logic.
   - Created `src/routes/authRoutes.js` for `/register`, `/login`, and protected `/profile` auth endpoints.
   - Created `src/middlewares/authMiddleware.js` to verify Bearer JWT tokens and attach the authenticated user to `req.user`.
   - Updated `src/models/User.js` so emails are normalized with lowercase and trim, and passwords are excluded from default query results.

8. Add frontend authentication pages and API mutation handling

   Command:

   ```bash
   cd client
   npm install @tanstack/react-query
   ```

   Installed dependencies:

   - `@tanstack/react-query`: Server-state library used to manage async API calls, loading states, success callbacks, and error handling for login and registration requests.

   Frontend setup changes:

   - Wrapped the React app with `QueryClientProvider` in `src/App.jsx`.
   - Created routes for `/login`, `/register`, and `/dashboard` using `react-router-dom`.
   - Redirected the root route `/` to `/login`.
   - Created `src/store/authStore.js` with Zustand to store the logged-in user and JWT token in local storage.
   - Created `src/pages/Login.jsx` to call `POST http://localhost:5000/api/auth/login`, save credentials, and navigate to the dashboard.
   - Created `src/pages/Register.jsx` to call `POST http://localhost:5000/api/auth/register`, save credentials, and navigate to the dashboard.
   - Created `src/pages/Dashboard.jsx` as the initial protected app area placeholder.

9. Add protected frontend studio and code editor

   Command:

   ```bash

   npm install @monaco-editor/react
   ```

10. Add OpenAI-powered backend code generation

   Command:

   ```bash
   cd server
   npm install openai
   ```

   Installed dependencies:

   - `openai`: Official OpenAI Node.js SDK used by the backend to call OpenAI models for AI-assisted code generation.

   Environment setup:

   - Added `OPENAI_API_KEY` to `server/.env` so the backend can authenticate OpenAI API requests without hardcoding secrets.

   Backend setup changes:

   - Created `src/controllers/aiController.js` with a protected `generateCode` controller.
   - Added a two-step AI generation flow: one model call creates an architectural blueprint, and another model call generates code from that blueprint.
   - Updated the user generation count after successful code generation.
   - Created `src/routes/aiRoutes.js` with `POST /generate`, protected by the JWT auth middleware.
   - Mounted AI routes under `/api/ai` in `index.js`.

   Frontend setup changes:

   - Updated `src/pages/Studio.jsx` to call `POST http://localhost:5000/api/ai/generate`.
   - Sent the saved JWT token in the `Authorization: Bearer <token>` header.
   - Replaced the simulated code generation result with the real backend response and displayed returned code in the Monaco editor.
   - Added error handling so failed generation requests show an error message in the editor.
